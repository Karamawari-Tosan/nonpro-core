# 縦書き書籍自動組版システム 制作マニュアル

Vivliostyleを使用した、Markdown原稿から縦書きPDFを生成するための制作フローおよび環境構築マニュアルです。

## 1. プロジェクト構成 (Standard Directory Structure)

新規プロジェクトを作成する際は、以下のディレクトリ構成を推奨します。

```text
Project_Root/
├── manuscript/          # [必須] 原稿用Markdownファイルを格納
│   ├── 01_Chapter.md    # 章ごとのファイル（ファイル名順に結合されます）
│   ├── 02_Chapter.md
│   └── colophon.md      # 奥付ファイル（ファイル名に「奥付」または「colophon」を含むものを自動認識）
├── scripts/             # [必須] ビルド用スクリプト
│   └── preprocess.js    # 前処理スクリプト（目次生成、縦書き処理、QRコード生成）
├── theme.css            # [必須] 書籍のデザイン設定（フォント、余白、ノンブルなど）
├── vivliostyle.config.js # [必須] Vivliostyle設定ファイル
└── package.json         # [必須] 依存パッケージ管理
```

## 2. 環境構築 (Setup)

### A. 自動セットアップ（推奨）
Antigravityのワークフロー機能を使用して、新規プロジェクトを最速で構築できます。

1. **コマンド実行**: チャット欄で以下のコマンドを入力します。
   ```text
   @antigravity /create_vertical_book [プロジェクト名]
   ```
   例: `/create_vertical_book 10_Projects/MyBook`

2. **完了**: 自動的にフォルダ作成、ファイルコピー、依存関係のインストールが行われます。すぐに原稿を書き始められます。

### B. 手動セットアップ
テンプレートを手動でコピーして開始する場合の手順です。

1. **テンプレートのコピー**: `00_Environment_Dev/Vivliostyle_Template` の中身を、新規フォルダにコピーします。
2. **依存パッケージのインストール**:
   ターミナルでコピー先のフォルダを開き、以下のコマンドを実行します。
   ```bash
   npm install
   ```

### C. プロジェクトの独立化（別リポジトリへの移行）
このプロジェクトは自己完結型（Standalone）であるため、フォルダごと移動しても動作します。Gitリポジトリとして独立させたい場合は以下の手順を行います。

1. **フォルダ移動**: プロジェクトフォルダ（例: `300_Takahashi_Tech_Doujin`）をデスクトップや別の場所へ移動します。
2. **Git初期化**: ターミナルでそのフォルダを開き、`git init` を実行します。
3. **ビルド確認**: フォルダ移動後も、そのまま `npm run build` が動作することを確認してください。

### D. Git管理について (.gitignore)
本テンプレートには、あらかじめ推奨設定の `.gitignore` が含まれています。以下のファイルは自動的にGit管理から除外されます。

- `node_modules/` (依存パッケージ)
- `.vivliostyle/` (ビルドキャッシュ)
- `book.pdf` (成果物)
- `.claude/`, `.obsidian/` (エディタ設定など)

`git init` を行った直後から、原稿と設定ファイルのみがクリーンに管理されるようになっています。

## 3. 原稿の準備 (Manuscript Preparation)

`manuscript` フォルダ内にMarkdown形式(`.md`)で原稿を配置します。

### ルールと機能
- **ファイル順序**: ファイル名順（昇順）に結合されます。`01_...`, `02_...` のように番号を振ることを推奨します。
- **見出し (TOC)**: `# 見出し1` (H1) で記述された箇所が、自動的に「目次」として抽出されます。
- **縦書き自動処理**: 半角英数字は自動的に縦中横（2桁まで）または正立（1桁）のクラスが付与されます。
- **改ページ**: ファイルの結合部には自動的に改ページが入ります。手動で入れたい場合は `<div style="break-before: page;"></div>` を記述してください。

### 特殊機能: QRコード生成
URLをQRコードとして紙面に掲載できます。

- **Markdownリンク形式**: `[リンクテキスト](https://example.com)` と記述すると、そのURLのQRコードが自動生成され、行内に配置されます。
- **直書きURL**: `https://example.com` とそのまま書いた場合は、QRコードは**生成されません**（テキストのみ表示）。

### 奥付 (Colophon)
奥付ファイル（例: `colophon.md`）には、以下のようにQRコード専用行を作ることができます。

```markdown
**発行所**: 株式会社プランノーツ http://plannauts.co.jp/
（↑ここはテキストのみ表示）

**電子版**: [ダウンロード](http://plannauts.co.jp/)
（↑ここはQRコードが生成される）
```

## 4. ビルドと出力 (Build)

PDFを作成するには、以下のコマンドを実行します。

```bash
npm run build
```

- 処理が完了すると、プロジェクトルートに `book.pdf` が生成されます。
- 中間生成物として `vivliostyle_manuscript/book_full.md` が生成されますが、これは自動生成ファイルなので編集しないでください。

## 5. デザイン調整 (Theming)

`theme.css` を編集することで、見た目を調整できます。

- **フォント**: 英数字用、日本語用のフォント設定
- **余白**: `@page` 設定内
- **QRコード**: `.qr-code-wrap` クラス
- **目次**: `.toc` クラス

---
**Troubleshooting**:
- **QRコードが出ない**: URLが `[text](url)` の形式になっているか確認してください。
- **ビルドエラー**: `node_modules` があるか、`npm install` したか確認してください。
