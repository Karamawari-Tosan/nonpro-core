## 2. 識別された課題

### 🔴 高優先度

#### 2.1 ルートディレクトリの整理
**問題点:**
- `PROJECT_IMPROVEMENT_ANALYSIS.md` がルートに配置されている
- 一時的な分析レポートであり、恒久的なドキュメントではない
- 今後も同様の分析ドキュメントが増える可能性がある

**推奨対応:**
- アーカイブディレクトリ（`90_Archive/` または `docs/archive/`）への移動
- または内容を適切なドキュメントに統合後、削除

#### 2.2 命名規則の不統一（99_Guide/）
**問題点:**
- `00_Setup_Guide.md` - 番号プレフィックス + アンダースコア + 英語
- `Antigravity セキュリティ・プライバシー 利用ガイドライン.md` - 番号なし + 日本語 + スペース

**影響:**
- ファイルのソート順が不明確
- 命名パターンの予測が困難

**推奨対応:**
- 統一ルールの策定と適用
- オプション1: `01_Security_Privacy_Guidelines.md`（英語形式）
- オプション2: `01_セキュリティプライバシーガイドライン.md`（日本語形式、スペース除去）

### 🟡 中優先度

#### 2.3 ドキュメント間のナビゲーション
**問題点:**
- README.mdから各ガイドへのリンクが部分的
- 99_Guide/ に統合インデックスがない
- 00_Context_Library/ にもインデックスがない（PROJECT_IMPROVEMENT_ANALYSIS.mdで提案済み）

**推奨対応:**
- `99_Guide/README.md` の作成（全ガイドのインデックス）
- `00_Context_Library/INDEX.md` の作成（既存提案の実装）

#### 2.4 メタデータの標準化
**問題点:**
- 一部のドキュメントにのみ更新日やステータスが記載
- フロントマター（YAMLメタデータ）の不在

**推奨対応:**
- ドキュメントテンプレートの作成
- メタデータヘッダーの標準化

## 3. 命名規則・配置ルール（提案）

### 3.1 ファイル命名規則

#### ガイド・マニュアル類（99_Guide/）
**形式:** `[番号]_[英語名].md`

- 例: `00_Setup_Guide.md`, `01_Security_Guidelines.md`
- 理由:
  - 番号によるソート順の明確化
  - 英語名による一貫性
  - アンダースコアによる可読性

#### コンテンツ・ナレッジ類（00_Context_Library/）
**形式:** 日本語可、意味重視

- 例: `コミュニティ3.0.md`, `Identity.md`
- 理由: コンテンツの本質を表現するため、日本語の使用を許可

### 3.2 ディレクトリ配置ルール

| ディレクトリ | 配置基準 | 例 |
|:---|:---|:---|
| `/` (ルート) | プロジェクト概要のみ | README.md, LICENSE |
| `99_Guide/` | ユーザー向けガイド・マニュアル | Setup, Security, Playbook |
| `00_Context_Library/` | AIエージェント参照用ナレッジ | 企業情報、戦略、用語集 |
| `80_Resources/` | 再利用可能なテンプレート・ツール | Vivliostyle, 画像素材 |
| `.agent/` | AIエージェント定義 | Skills, Workflows |
| `90_Archive/` (新設) | 古いドキュメント・一時ファイル | 過去の分析レポート |

### 3.3 メタデータ標準

全 `.md` ファイルに以下のフロントマターを推奨:

```yaml
---
title: "ドキュメントタイトル"
last_updated: 2026-03-22
status: Active  # Draft | Active | Archived
category: Guide  # Guide | Knowledge | Resource | Playbook
tags: [setup, security]
---
```
