## 概要
手動で新規フォルダを作成し、VS Code ワークスペースに `nonpro-core` と共に追加した状態から開始する、プロジェクト初期設定の標準手順です。
Git初期化、各種ドキュメント（README.md / AGENTS.md）の生成、GitHubリポジトリの作成、および初期イシューの起票までを自動化します。

## 手順

1. **プロジェクトの基本情報の確認**
   AIはユーザーに対して以下の項目を対話形式で確認・提案してください。
   ※前提として、現在AIが作業しているディレクトリをプロジェクトのルートと見なします。
   - **プロジェクト名**（リポジトリ名やドキュメントタイトルに使用）
   - **プロジェクトの目的・ゴール**
   - **想定ターゲット・ペルソナ**
   - **主なアウトプットの種類**（例：記事、ポッドキャスト、イベント、Webアプリなど）
   - **マイルストーン・スケジュール**
   - **AIと人間の役割分担**（AIに任せること／人が判断すること）

2. **Git初期化と .gitignore の作成**
   以下のコマンドを実行し、標準的な無視設定を適用します。
   - `git init` の実行
   - `.gitignore` の作成。以下の項目を必ず含めること。
     - **AI/ツール関連:** `.claude`, `.gemini`, `.agent/scratch/`, `.gemini/`
     - **環境・秘匿情報:** `.env`, `*.local`, `config.json`（秘密情報を含む場合）
     - **OS関連:** `Thumbs.db`, `.DS_Store`
     - **開発関連:** `node_modules/`, `__pycache__/`, `.vscode/`, `*.code-workspace`
   
   *エラーハンドリング:*
   Gitコマンドが失敗した場合は、権限の問題や既に初期化済みでないかを確認し、ユーザーに状況を報告してください。

3. **README.md の作成**
   手順1で確認した内容に基づき、プロジェクトルートに `README.md` を作成します。
   構成案：
   - **背景**
   - **目的・ゴール**
   - **ターゲット・ペルソナ**
   - **アウトプットのイメージ・スコープ**
   - **スケジュール・マイルストーン**
   - **AIと人間の役割分担**
   ※タスク管理はGitHubイシューで行うため、READMEにTODOリストは含めないでください。

4. **プロジェクト用 AGENTS.md の作成**
   プロジェクト固有の作業ルールを `AGENTS.md` に定義します。
   - プロジェクトの簡潔な説明（1〜3行）
   - 「必ず README.md の内容（目的・役割分担等）を前提として動作すること」という指示
   - 固有の制約や推奨ツール（あれば）
   - 既存の `.agent/rules/` 等がある場合はその整合性確認

5. **GitHub リポジトリの作成と初期イシューの起票**
   以下の手順で GitHub 連携を行います。
   - **認証確認:** `gh auth status` を実行し、未ログインであればユーザーに `gh auth login` を促してください。
   - **初回コミット:** `git add .` および `git commit -m "feat: initial commit"` を実行。
   - **リポジトリ作成:** 
     `gh repo create plannauts/<プロジェクト名> --private --source=. --remote=origin --push --description "<日本語でプロジェクトの概要>"`
   - **初期イシューの作成:**
     READMEのマイルストーンや目的から、すぐに着手すべき具体的なタスクを AI が 2〜3 個考案し、以下の形式（--title, --body）で起票してください。
     *例:* `gh issue create --title "タスク名" --body "詳細な内容・背景・期待されるアウトプット"`

6. **完了報告**
   すべての処理が終わったら、以下をまとめて報告してください。
   - 作成・更新したファイルの一覧
   - 作成された GitHub リポジトリの URL
   - 起票した初期イシューの内容と、次に行うべき操作の提案
   
   *エラー報告:*
   途中でエラーが発生した場合は、どの手順まで完了し、何が失敗したかを明示して報告すること。
