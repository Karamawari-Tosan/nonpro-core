# イベント終了後の参加者向けメッセージ送信フロー

イベント終了後に参加者への感謝を伝え、ノンプロ研コミュニティへの誘導および次回の告知をセットで行う標準的なワークフローです。CCO（最高コミュニティ責任者）の「庭師」としての視点を大切にし、単なる事務連絡ではなく「学びの継続」を提案します。

---

## 1. 原稿の作成 (Copywriting)

イベントごとに、プロジェクトフォルダ配下に `04_FollowUp_Message.md` を作成して原稿を準備します。

### 1-1. 構成要素 (Standard Template)
以下の順序で構成します。

1.  **参加への感謝:** イベントへの出席に対するお礼。
2.  **イベントの振り返り:** 当日のハイライトや「持ち帰ってほしい価値」を再提示する。
3.  **ノンプロ研（コミュニティ）の案内:**
    *   **AIエージェント実験室:** 「自分で手を動かせる場所」としての提案。
    *   **アーカイブ動画:** 過去の知見にアクセスできるメリット。
    *   **入会リンク:** https://plannauts.co.jp/nonproken/
4.  **次回イベントの告知:**
    *   次回の日時、テーマ。
    *   PeatixページURL。
5.  **特典案内（必要に応じて）:** 会員向けクーポンコードの存在など。

### 1-2. トーン＆マナー
*   **庭師のアプローチ:** 参加者を「一人にしない」「共学の仲間として迎える」親しみやすく丁寧な口調。
*   **心理的安全性:** 「失敗しても大丈夫」「まずは実験してみましょう」という前向きなメッセージ。

---

## 2. メッセージ送信手順 (Peatix Operations)

作成した原稿をPeatix経由で参加者に送信します。

1.  **Peatix管理画面にログイン:** 対象のイベントを選択します。
2.  **「参加者へのメッセージ」を選択:**
    *   「メールを送信」または「メッセージを送信」機能を使用します。
3.  **原稿の流し込み:**
    *   準備した Markdown 原稿を貼り付けます。
    *   **注:** Peatixのメッセージ機能はプレーンテキストが基本のため、Markdownの記号（#や*）は、太字などの装飾の代わりに区切り線（━━━━━━━━）や箇条書き（■、・）として読みやすく調整します。
4.  **テスト送信（推奨）:** 自分宛てにテスト送信し、リンクが正しく機能するか、改行が読みやすいかを確認します。
5.  **一斉送信:** 参加者全員（キャンセル者を除く）に送信します。

---

## 3. 次のアクション (Follow-up)

1.  **Slackでの反応:** Slack上の関連チャンネル（`#general` 等）でメッセージ送信を報告し、補足情報があれば共有します。
2.  **アンケート確認:** 寄せられたアンケート結果を元に、次回のコンテンツやメッセージ内容を微調整します。
